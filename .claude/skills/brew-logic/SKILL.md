# /brew-logic — コーヒー抽出補正ルールエンジン

## 概要
コーヒー抽出補正のルールベースロジック専用スキル。
入力パラメータから味の偏りに対する調整提案を生成する。AI APIは使わない。

## 起動条件
ユーザーが `/brew-logic` と入力したとき、または抽出ルールの追加・修正・確認を依頼されたとき。

## 入力パラメータ

| パラメータ | 単位 | 範囲例 |
|-----------|------|--------|
| 豆量（dose） | g | 10-30g |
| 湯温（temperature） | ℃ | 80-100℃ |
| 抽出時間（time） | 秒 | 30-300秒 |
| 挽き目（grindSize） | 段階 | 1(極細)-10(極粗) |
| 湯量（waterAmount） | ml | 100-500ml |

## 味の偏り分類

| 偏り | 説明 |
|------|------|
| 苦すぎる（bitter） | 過抽出の傾向 |
| 酸っぱすぎる（sour） | 未抽出の傾向 |
| 薄い（weak） | 濃度不足 |
| 濃すぎる（strong） | 濃度過多 |
| えぐみ（astringent） | 微粉・高温による雑味 |

## 補正ルール体系

### 基本ルール（苦い → 過抽出を緩和）
- 湯温を2-3℃下げる
- 挽き目を1段階粗くする
- 抽出時間を10-15秒短くする

### 基本ルール（酸っぱい → 抽出を促進）
- 湯温を2-3℃上げる
- 挽き目を1段階細かくする
- 抽出時間を10-15秒長くする

### 基本ルール（薄い → 濃度を上げる）
- 豆量を1-2g増やす
- 湯量を20-30ml減らす
- 挽き目を1段階細かくする

### 基本ルール（濃い → 濃度を下げる）
- 豆量を1-2g減らす
- 湯量を20-30ml増やす
- 挽き目を1段階粗くする

### 基本ルール（えぐみ → 雑味を除去）
- 湯温を5℃下げる
- 挽き目を1段階粗くする（微粉削減）
- 最初の蒸らし後にかき混ぜない

## ルール追加時の矛盾チェック

新しいルールを追加する際は以下をチェック：

1. **方向性矛盾**: 同じ味の偏りに対して逆方向の調整がないか
   - 例：「苦い→湯温上げる」は既存ルール「苦い→湯温下げる」と矛盾
2. **循環矛盾**: ルールAの出力がルールBのトリガーになり、ルールBの出力がルールAのトリガーになるパターン
3. **範囲逸脱**: 調整後のパラメータが有効範囲を超えないか
4. **複合効果**: 複数の調整が同時に適用された場合の相互作用

## 注意事項
- AI APIは絶対に使わない。すべてルールベースで処理する
- ルールは `lib/brew-rules/` または `utils/brew-logic/` に実装する
- ルール追加時は必ず矛盾チェックを実行する
- 初心者向けなので、一度に変更するパラメータは最大2つまでに制限する
